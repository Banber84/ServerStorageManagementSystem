# 文档总览

项目文档按用途分为三类，功能说明和测试记录不混写。

## 设计与功能

[`design/`](design/README.md) 保存系统架构、API、数据库、Agent 和运行机制。
需要理解“系统为什么这样工作”时从这里开始。

## 部署与操作

[`deployment/`](deployment/README.md) 保存安装、配置、`ssmsctl` 命令、节点接入、
用户同步、Gateway 和验收方案。需要执行日常运维命令时从这里开始。

## 测试报告

[`reports/`](reports/README.md) 只保存已经执行过的测试过程、终端现象、问题修复
和结论。报告中的历史 IP 与命令不作为当前部署入口。

## 维护规则

- 功能原理、参数定义和当前推荐操作写入 `design/` 或 `deployment/`。
- 具体测试日期、环境、终端输出和结果写入 `reports/`。
- 同一条日常命令只在功能文档维护，报告通过链接引用。
- 当前推荐 CLI 入口是 `ssmsctl`，底层脚本只用于实现和故障排查。
